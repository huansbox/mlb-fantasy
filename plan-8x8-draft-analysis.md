# 8×8 H2H 選秀分析計畫

> **For agentic workers:** REQUIRED: Use superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 從現有 ADP 資料出發，產出一份針對 8×8 H2H 格式的實戰選秀排名與逐輪行動指南

**Architecture:** 三層遞進分析——先對每位球員做格式類別貢獻評分（地基），再計算跨守位替代價值 VOR（結構），最後疊加 ADP 套利標記產出實戰 cheat sheet（成品）

**輸入資料：**
- `2026 FanGraphs Fantasy Baseball ADP 資料.md`（ADP + 守位 + $ 值）
- `2026 守位稀缺性分析 - 內野與捕手篇.md`（定性判斷）
- `*-claude.md` / `*-gemini.md`（兩份策略報告，作為交叉參考）

**輸出文件：**
- `分析1-格式類別貢獻評分.md`
- `分析2-跨守位替代價值VOR.md`
- `分析3-實戰選秀排名與行動指南.md`

**聯賽設定備忘：**
- 12 隊，H2H Categories，蛇形選秀，22 輪
- 名單：8 野手 (C/1B/2B/3B/SS/OF×3) + 2 UTIL + 4 SP + 2 RP + 3 P + 板凳/IL/NA
- 打者 8 項：R, HR, RBI, SB, BB, K(負向), AVG, SLG
- 投手 8 項：W, SV, BB(負向), K, HLD, ERA, WHIP, QS
- 每週最低 65 IP，每週最多 6 次異動

## Task 1：打者格式類別貢獻評分

**目的：** 對前 150 ADP 內的所有打者，逐人標記八項類別的正負貢獻，產出格式淨貢獻分數

**輸入：** ADP 資料中的打者名單
**輸出：** `分析1-格式類別貢獻評分.md`

### 評分框架

每位打者在八項類別中標記為：
- **+2** 菁英正向（該類別前 10%）
- **+1** 正向貢獻（高於平均）
- **0** 中性
- **-1** 負向拖累（低於平均）
- **-2** 嚴重拖累（該類別後 10%）

格式淨貢獻分 = 八項加總（範圍 -16 到 +16）

### 評分所需數據指標

需要查找或估算每位打者的：
- K%（三振率）→ 決定 K 欄評分
- BB%（保送率）→ 決定 BB 欄評分
- ISO / barrel rate → 輔助 HR 和 SLG 評分
- Sprint speed / SB 預測 → 決定 SB 評分
- 預測 AVG → 決定 AVG 評分
- 預測 SLG → 決定 SLG 評分
- 預測 R、RBI → 決定計數數據評分

### 步驟

- [ ] **Step 1：定義評分閾值**
  - 根據 2025 全聯盟數據，設定每項類別的 +2/+1/0/-1/-2 分界線
  - 例：K% < 12% → +2，12-18% → +1，18-22% → 0，22-28% → -1，> 28% → -2
  - 將閾值表寫入文件開頭作為參考

- [ ] **Step 2：搜集打者進階數據**
  - 從 FanGraphs 或 Statcast 取得前 150 ADP 打者的 K%、BB%、barrel rate、sprint speed 等
  - 若無法逐人抓取，使用 2025 實際數據 + 2026 預測系統（Steamer/ZiPS）作為基底

- [ ] **Step 3：前 50 ADP 打者評分**
  - 逐人填寫八項評分 + 淨貢獻分
  - 標記格式贏家（淨分 ≥ +8）和格式陷阱（淨分 ≤ +2 但 ADP 在前 50）

- [ ] **Step 4：ADP 51-150 打者評分**
  - 同上流程
  - 特別關注「標準聯賽被低估但格式受益」的球員（如 Perdomo、Bohm）

- [ ] **Step 5：產出打者評分總表**
  - 按格式淨貢獻分排序
  - 標註與 ADP 排名的落差（格式排名 vs ADP 排名）
  - 附上前 10 名格式贏家和前 10 名格式陷阱的摘要

## Task 2：投手格式類別貢獻評分

**目的：** 同 Task 1 邏輯，對 SP 和 RP 做八項投手類別評分

**輸入：** ADP 資料中的投手名單
**輸出：** 追加至 `分析1-格式類別貢獻評分.md`

### SP 評分考量
- QS 需要關注「每場平均投球局數」和「投滿 6 局機率」
- W 受隊友打線影響，需考慮所屬球隊預測勝場
- BB 作為負向類別，控球差的投手雙重受罰（BB + WHIP）
- K 欄需要高三振率投手

### RP 評分考量
- 區分 CL（主攻 SV）和 SU（主攻 HLD）
- 在此格式下 SU 和 CL 各貢獻一個獨立類別，價值應接近等值
- 評估 RP 時額外加入「角色穩定度」因素（球隊爭冠程度、競爭者數量）

### 步驟

- [ ] **Step 6：SP 評分（前 30 名 SP）**
  - 八項評分：W, SV(通常為 0), BB, K, HLD(通常為 0), ERA, WHIP, QS
  - 特別標記「格式溢價 SP」：高 QS + 低 BB + 高 K 的組合（如 Webb、Kirby、Woo）

- [ ] **Step 7：RP 評分（前 30 名 CL + 前 20 名 SU）**
  - CL 評分重點：SV 產量 + 比率貢獻
  - SU 評分重點：HLD 機會 + K/9 + ERA/WHIP
  - 標記「格式溢價 RP」：高 HLD + 菁英比率的中繼投手

- [ ] **Step 8：SP vs RP 跨角色比較**
  - 在此格式下，一名菁英 SU（如 Whitlock：24 HLD + 2.25 ERA）vs 一名中段 SP（如 Glasnow：高 K 但傷病風險），誰的格式淨貢獻更高？
  - 產出 SP/RP 混合的投手格式評分排名

## Task 3：跨守位替代價值（VOR）計算

**目的：** 將 Task 1-2 的格式評分轉化為跨守位可比較的統一排名

**輸入：** `分析1-格式類別貢獻評分.md`
**輸出：** `分析2-跨守位替代價值VOR.md`

### VOR 計算方法

1. 定義每個守位的「替代水平」球員（12 隊聯賽中，該守位第 13-15 名的球員）
2. 每位球員的 VOR = 該球員格式淨貢獻分 - 同守位替代水平球員的格式淨貢獻分
3. VOR 越高 = 越值得提前選

### 各守位替代水平預估

| 守位 | 先發需求 | 替代水平（約第 N 名） | 預估替代球員 |
|------|---------|---------------------|------------|
| C | 12 | 第 13 名 C | 待 Task 1 確認 |
| 1B | 12-15 | 第 15 名 1B | 待確認 |
| 2B | 12 | 第 13 名 2B | 極差（稀缺位） |
| SS | 12 | 第 13 名 SS | 待確認 |
| 3B | 12 | 第 13 名 3B | 待確認 |
| OF | 36-40 | 第 40 名 OF | 尚可（深度位） |
| SP | 48-60 | 第 55 名 SP | 待確認 |
| RP | 24-36 | 第 30 名 RP | 待確認 |

### 步驟

- [ ] **Step 9：確定各守位替代水平**
  - 根據 Task 1-2 的評分結果，找出每個守位的替代水平球員及其格式淨貢獻分

- [ ] **Step 10：計算全員 VOR**
  - 每位球員的 VOR = 格式淨貢獻分 - 守位替代水平分
  - 多守位資格球員取最高 VOR 的守位

- [ ] **Step 11：產出跨守位統一排名**
  - 按 VOR 排序，不分守位
  - 這就是「如果只看格式價值，你應該按這個順序選」的理想排名
  - 標註每位球員的主要守位和多守位資格

## Task 4：ADP 套利分析與實戰選秀指南

**目的：** 比對 VOR 排名與市場 ADP，找出套利機會，產出逐輪行動指南

**輸入：** `分析2-跨守位替代價值VOR.md` + ADP 資料
**輸出：** `分析3-實戰選秀排名與行動指南.md`

### 步驟

- [ ] **Step 12：計算 ADP 落差**
  - 每位球員：VOR 排名 vs ADP 排名 → 落差值
  - 正落差（VOR 排名 >> ADP 排名）= 被市場低估 → 買入
  - 負落差（VOR 排名 << ADP 排名）= 被市場高估 → 避開

- [ ] **Step 13：標記套利目標**
  - 🟢 格式溢價（VOR 排名比 ADP 高 30+ 位）：這些球員你可以比市場晚 1-2 輪選到
  - 🔴 格式陷阱（VOR 排名比 ADP 低 30+ 位）：這些球員在你的格式下不值那個 ADP
  - 🟡 公平定價（落差 < 15 位）：按 ADP 正常選即可

- [ ] **Step 14：逐輪行動指南**
  - 按 22 輪結構，每輪列出：
    - 該輪的首選目標（2-3 人）
    - 備選方案（2-3 人）
    - 避開名單（1-2 人）
    - 守位檢查提醒（「如果到這輪還沒選 2B，必須在此輪解決」）
  - 融入守位稀缺性分析的結論

- [ ] **Step 15：特殊情境決策樹**
  - 幾個常見的選秀分叉點：
    - 「第 3 輪：Marte (2B) vs Contreras (C) vs SP1？」
    - 「第 5-6 輪：開始拿 SP 還是繼續補打者？」
    - 「第 10-12 輪：拿 CL 還是繼續等用 SU 填 HLD？」
  - 提供每個分叉的建議及理由

- [ ] **Step 16：最終審查與交叉驗證**
  - 用模擬選秀的方式走一遍 22 輪，檢查：
    - 是否滿足所有守位需求
    - SP 數量是否足以跨過每週 65 IP
    - 是否有至少 1 CL + 2 SU 覆蓋 SV 和 HLD
    - 整體陣容的 K 欄是否健康（打者端）
    - 整體陣容的 BB 欄是否健康（投手端）

## 執行順序與依賴關係

```
Task 1（打者評分）──┐
                    ├──→ Task 3（VOR）──→ Task 4（實戰指南）
Task 2（投手評分）──┘
```

- Task 1 和 Task 2 可平行處理
- Task 3 依賴 Task 1 + 2 的完成
- Task 4 依賴 Task 3 的完成

## 風險與限制

1. **數據精度**：如果無法取得每位球員的即時進階數據，部分評分需依賴歷史趨勢和預測系統推估，會有誤差
2. **ADP 時效性**：FanGraphs ADP 非 Yahoo 專屬，實際 Yahoo ADP 會有差異。使用者應在選秀前用 Yahoo App 上的最新 ADP 做最後校正
3. **新秀不確定性**：Caminero、Kurtz、Roman Anthony 等新秀缺乏大聯盟數據，評分高度依賴預測系統
4. **季中變動**：此分析基於開季前資料，傷病、交易、角色變動會改變實際價值
