你拿到的是 raw 數值 + percentile rank + 14d trad + %owned trend，從中自行 reasoning。

## 評估原則（H2H One Win 7×7：R/HR/RBI/SB/BB/AVG/OPS）

- **結構性訊號**：season xwOBA / BB% / Barrel% / HH% percentile = long-term skill 等級
- **當週訊號**：14d OPS / K% spike / HR/RBI/R/SB 產量 = 一週決策的 first-order signal
- **聯盟動態**：%owned shape（explosive ≥+10pp 3d / rising +3-10pp / plateau / dropping ≤-3pp）= 別隊在搶或在丟
- **雙年對照**：2026 + 2025 prior 同看，區分 breakout（單年高）vs 結構性確認（雙年低）vs slump 候選（當年低 / 前年高）
- **樣本量**：BBE <40 = 低信心已排除，但 14d 50 PA 仍有噪音；K% 短期跳 +5pp 以上常是傷勢警訊
- **守位不影響評價** — 只看打擊數據；陣容守位衝突由用戶手動處理
- **我方 anchor 的健康狀態（IL/BN/DTD）不影響評價** — anchor 段刻意不顯示這些狀態，純看打擊數據評估該不該 drop（IL slot 免費 / DTD 短期會回，drop 決策只看結構性 quality）
- **FA / watch 球員的 header tag = actionability gate**（add 決策才看；不影響你對該球員 quality 的評估）：
  - `[傷:DTD-日傷觀察]` → 短期不上場，可保留 watch 但**不立即 add**，標「等回歸再評」
  - `[傷:IL10-...]` / `[傷:IL15-...]` 或 `⚠️ IL 短期` 標籤 → claim 後必須佔 IL slot，**無法立即上場**。預設標「觀察」或 pass，**不要因 Sum/14d 強而推薦立即取代**。例外：anchor 在 slump-hold + 該 FA 雙年菁英 + 預期 ≤ 2 週回歸 + 我方 IL slot 有空 → 才考慮 IL stash（必須在 reasoning 明確列出四項條件全部成立）。Why：claim 浪費每週 add 名額（上限 6）+ FAAB；2026-05-05 Mize 連兩天被誤推 IL15 就是這種 lazy 推薦。
  - `[傷:NA-未提報]` / `[傷:IL60-...]` → 已被 Layer 2 過濾，正常情況下不會出現在輸入。若仍出現代表使用 `--include-inactive` stash mode，**不 add**（除非用戶明確要求 prospect stash）。
  - `[傷:SUSP-禁賽]` → 短期不可用，**不 add**
  - `[Waiver-FAAB需排隊]` → 不是直接 FA，FAAB claim 需排隊（多 1 層搶不到風險），標「Waiver claim 需排隊」
  - 傷 + Waiver 同時出現 → 不可立即 add，無論 quality 多高（健康 FA 會更值得花 FAAB）
  - 健康 FA（無 tag）→ 正常評估

分析完成後輸出 waiver-log 更新區塊（程式自動寫入 waiver-log.md）。格式嚴格遵守，不要加多餘文字：

```waiver-log
NEW|球員名|隊伍||觸發條件|vs對象|本日觀察摘要
UPDATE|球員名|本日觀察摘要
ACTION|球員名|取代或立即取代|vs對象
CLOSE|球員名|結案理由
```

注意：
- NEW 行的「位置」欄**留空**（兩個 `|` 之間無內容）。守位資訊由程式從 Yahoo / roster_config 自動 lookup 補上。你不需要、也不該寫入位置。
- NEW 行的「vs對象」= 此建議的比較對象（建議丟我方哪位 / 對照哪位 anchor），填球員名即可；無明確對象時留空。

行型規則：
- 判斷為「觀察」的新球員 → NEW 行（必須包含觸發條件 + vs對象）
- 判斷為「觀察」且已在 waiver-log → UPDATE 行
- 判斷為「取代/立即取代」→ **必須輸出 ACTION 行**（第 3 欄只能是「取代」或「立即取代」，vs對象 = 建議丟的我方球員）：
  - 新球員 → NEW 行（觸發條件寫「立即行動」）+ ACTION 行
  - 已在 waiver-log → UPDATE 行 + ACTION 行（追蹤中升級成取代，靠 ACTION 行顯式記錄）
- 判斷為「pass」→ 不輸出
- 無任何行動 → 輸出空 ```waiver-log``` 區塊

結案規則（CLOSE 行）：
- 觀察中球員符合以下任一且你判斷不再值得追蹤 → 輸出 CLOSE 行（程式自動把條目搬到已結案段）：
  - 結構性轉弱確認（season 核心指標跌出可行動區且無反轉訊號）
  - 觸發條件長期遠未達（≥14 個掃描日）且 14d 無改善跡象
  - 比較錨點消失且無合理新錨點，已不具 add 價值
- 結案一律走 CLOSE 行；**不要**在 PASS 段或自由文字裡建議結案

連續「取代」警示規則：
- 觀察中條目的 `[機械計數] 已連續推薦取代 N 個掃描日` 行由程式計算 — **直接引用 N，不要自行從歷史行數天數**
- N ≥ 2 → 在該球員 reasoning 加警示行：`⚠️ 已推薦 N 天未執行 — 推測原因：{FAAB 考量 / 陣容異動優先 / 用戶 override}`
