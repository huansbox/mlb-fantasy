## Part 2: 數據完整性對比（WebFetch vs roster_stats.py）

### Savant 指標（xwOBA / HH% / BBE）：完全一致

所有打者、投手的 xwOBA、HH%、BBE 兩邊數字完全吻合。WebFetch AI 解析 CSV 無誤差。

### MLB API 傳統數據：小幅時差

| 球員 | 手動 PA/OPS | 腳本 PA/OPS | 原因 |
|------|------------|------------|------|
| Altuve | 42 / 1.048 | 43 / 1.106 | 抓取時間差（比賽進行中即時更新） |
| Albies | 42 / .819 | 43 / .799 | 同上 |

其他 10 位打者 + 全部投手傳統數據完全一致。

### 差異 1：Barrel% = 0 被腳本 fmt() 吃掉（已修）

手動抓到 0% Barrel（Machado / Butler / R. Garcia / Whitlock / Messick / Bassitt），腳本原本顯示 `—`。
原因：`roster_stats.py` 的 `fmt()` 把 `val == 0` 當缺值。已修正為只檢查 `val is None`。

### 差異 2：R. Garcia xERA / xwOBA

| 來源 | xERA | xwOBA |
|------|------|-------|
| 手動 (expected_statistics CSV) | 未找到 | 未找到 |
| 腳本 | 4.74 | .338 |

腳本透過 `fetch_savant_for_pitchers()` 用不同合併邏輯找到了 R. Garcia。腳本資料覆蓋率更好。

### 差異 3：腳本額外功能

百分位標記（>P90 / <P25 等）、QS 計算（從 game log 算）、前一年自動對照 — 手動版都沒有。

### Savant 更新頻率確認

Bassitt game log 顯示 2 場：3/30 + 4/5（美東）。台灣 4/6 時數據已包含 4/5 比賽。
確認：**Savant 和 MLB API 在比賽結束後當天即更新**。

### 結論

腳本資料更完整也更可靠，後續分析直接用腳本輸出。
